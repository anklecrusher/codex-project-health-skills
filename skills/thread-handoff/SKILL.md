# Thread Handoff

## Use This Skill For

- Compressing long tasks into a transferable handoff.
- Preparing a new thread to continue work without inheriting noise.
- Capturing what is done, what is blocked, what is verified, and what must happen next.
- Turning a drifting or overgrown conversation into a restartable package.

## Core Workflow

1. Identify the real goal and the reason the handoff is needed.
2. Summarize the current state in plain language.
3. Capture completed work, open risks, blockers, and verification status.
4. Separate hard facts from assumptions and unanswered questions.
5. Emit a compact handoff package that the next thread can use immediately.

## Handoff Format

Include these sections when relevant:

- Goal
- Current state
- Completed work
- Open risks
- Verification status
- Next safest step
- What the next thread can ignore

## Guardrails

- Keep the handoff concise and executable.
- Do not restate all background unless it changes the next move.
- Do not claim completion without verification evidence.
- Do not broaden scope beyond the transfer point unless the user asks.

## Outputs

- A ready-to-send handoff note.
- A short judgment on whether a new thread is justified.
- A compact recommendation for the next thread's starting point.
